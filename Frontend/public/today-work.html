5. ⚖ Medico-Legal Cases — new module

new feature 5/5 audit clean

Brand-new MLC module. Doctors can record medico-legal cases two ways: (a) External — patient already has an MLC cut elsewhere, doctor just captures basic identifiers + a local MLR; (b) Internal — fresh MLC with full incident, injuries, opinion. Every MLC gets a unique MLR number that stamps every produced patient document.

🔢 MLR number generator

<2-letter-prefix><4-digit-seq> — e.g. RK0001. Prefix is derived from the doctor's name:

  • Rajesh Kumar → first letter of first + last name → RK
  • Sandeep (single name) → first + last letter → SP
  • If natural prefix is taken, walks a deterministic candidate list until a free pair is found
  • Globally unique — first 2 letters identify exactly one doctor in the hospital
  • Pinned for life via Doctor.mlcPrefix
  • Sequence atomic via $inc

🏷 Auto-stamping

Once an MLC is cut, the patient is flagged isMLC: true and the mlcNumber is set. The reusable <MLCAutoStamp uhid={...} /> component drops in to any patient document page and shows:

  • Watermark seal (circular red MLR seal, top-right, prints on every page)
  • Print banner (in-flow red strip in the document header)

Already wired into Discharge Summary + OPD print.
